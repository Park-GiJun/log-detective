## HIGH (9건)

### Architecture (강현수)
1. **Filter 이중 등록 위험** — `ApiKeyAuthenticationFilter` 가 `@Component` 로 등록되면 Spring Boot 의 `ServletContextInitializer` 가 글로벌 FilterChain 에 자동 등록. SecurityConfig 의 `addFilterBefore` 와 함께 **두 번 호출** 가능. `FilterRegistrationBean` 으로 글로벌 등록 비활성화 또는 `@Component` 제거 후 SecurityConfig 에서 `@Bean` 으로만 등록.

### Security (박서진)
2. **Length leak in MessageDigest.isEqual** — 두 byte[] 길이 다를 시 즉시 false 반환 → 키 길이 timing leak. 고정 길이 SHA-256 해시 비교로 정규화.
3. **Multi-key rotation + rate-limit + lockout 부재** — 단일 정적 키 + brute-force 방어 없음. Redisson 기반 IP/키 단위 카운팅 + `List<String>` expectedApiKeys 권장.
4. **Actuator 엔드포인트 노출 (defense in depth)** — `management.endpoints.web.exposure.include` 명시 제한 부재. `env`, `beans`, `heapdump` 등 불필요한 엔드포인트 제거.

### Performance (이도윤)
5. **expectedApiKey.toByteArray() 핫패스 호출** — 매 요청마다 UTF-8 인코딩 + byte[] 할당. `init` 블록에서 1회 캐싱.

### Testing (김태현)
6. **apiKey blank 분기 테스트 누락** — `apiKey.isNotBlank()` 가드의 회귀 검출 불가.
7. **401 E2E 시나리오 부재** — 헤더 누락/잘못된 키로 호출 시 401 + Outbox 0건 검증 없음.

### Domain (윤지아)
8. **design.md 인증 정책 절 부재** — service-to-service 인증 모델 명문화 누락. 보안 결정이 코드 KDoc 에만 있어 휘발 위험.

### Quality (정하은)
9. **permitAll 경로 이중 하드코딩** — Filter `shouldNotFilter` 와 SecurityConfig `requestMatchers` 가 같은 경로를 2곳에 정의. 한쪽만 수정 시 동기 깨짐.
