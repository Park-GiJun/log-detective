## CRITICAL 이슈 (즉시 처리)

### 1. 컴파일 결함 — ChannelType 3중 불일치
- `ChannelType.kt:3-5` enum: **ES, KAFKA, CACHE**
- `OutboxPublisher.kt:47` when 분기: **ES, KAFKA, FILE, OTHERS**
- `V2__create_outbox_messages.sql:6` CHECK: **ES, KAFKA, FILE, OTHERS**
- `FILE`, `OTHERS` 는 enum 미정의 → **빌드 실패**. 더불어 enum 의 `CACHE` 는 when 에서 누락 (비-exhaustive).
- 단위 테스트 1건만 있었어도 즉시 잡혔을 결함이 머지된 점은 **CI 빌드 게이트 부재** 신호.

### 2. @Transactional 내 외부 IO — 이슈 #25 미해결
`OutboxPublisher.pollAndDispatch` 가 `@Transactional` + 100건 직렬 처리 + ES HTTP + Kafka send 를 한 트랜잭션에 묶음. 트랜잭션 길이 = Σ(ES + Kafka latency). HikariCP 고갈 + FOR UPDATE 락 점유로 다른 인스턴스 폴링 차단. **#25 의 결함이 위치만 Handler→Publisher 로 이동**.

### 3. 처리량 근본 미달 (10K TPS 목표의 1/100)
100 batch × 1s poll = 6K msg/min. ES `_bulk` API 미사용, Kafka producer batching 미사용, dispatch 직렬 처리.

### 4. 단위 테스트 0건
- Handler: ingest 1건 → outbox 2건(ES/KAFKA), batch N건 → 2N건, destination/payload/aggregateId 검증 0건
- Publisher: 백오프 5/10/20s, MAX_ATTEMPTS=5→DEAD, 미지원 채널, fetchPending=empty, payload 손상 0건

### 5. fetchPending 쿼리 결함 — FAILED 영구 미재시도
SQL 이 `status = 'PENDING'` 만 필터. `markFailed` 후 status=FAILED 가 된 행은 영원히 재시도 안 됨 → 백오프(5s × 2^n) 의미 자체가 무력화. **Outbox 의 보장 의미가 깨지는 회귀**.

## HIGH 이슈

### Architecture
- **OutboxPublisher 위치** (`infrastructure/scheduler/`) — CLAUDE.md 컨벤션 위반, `infrastructure/adapter/in/scheduler/` 로 이동 필요
- **inbound 어댑터가 outbound port 직접 호출** — `DispatchOutboxUseCase` (port.in) 신설 필요
- **Application Handler 의 인프라 누수** — ObjectMapper, ES 인덱스 네이밍(`logs-yyyy.MM.dd`) 이 application 계층에 위치

### Security
- **payload PII 평문 보관** (ip, userId, attributes 포함). PUBLISHED/DEAD retention 정책 부재 → GDPR 위반 소지
- **이슈 #26 (입력 검증) 미해결 상태**에서 Outbox 도입 — 미검증 페이로드가 그대로 outbox/DB/ES 로 흘러감

### Performance
- **KafkaTemplate.send() Future 미확인** → broker ack 실패 시 markPublished 후 데이터 유실 (이전 리뷰 반복 패턴 #4)
- **write amplification x3** — Handler 가 동일 payload 두 번 직렬화 + outbox 2N건 INSERT
- **`(status, next_attempt_at)` 일반 인덱스** — partial index (`WHERE status='PENDING'`) 권장

### Code Quality
- **패키지 camelCase** (`logEvent/`, `outbox/`) — Kotlin 컨벤션 위반, all-lowercase 권장
- **ES 인덱스 명명규약 위치** — Handler → ES adapter 로 이동

### Domain
- **ES 인덱스 키 UTC 고정** — KST 운영 정책 미정 (이슈 #31 와 충돌). R004(00:00~05:00 KST 룰), R001(5분 BruteForce 자정 경계) 정확성 영향
- **백오프 지연 (4회 실패 시 80s+)** — R001 5분 윈도우 밖으로 밀려 미탐 위험. 룰별 SLA 미정의
- **통합 테스트 부재** — Testcontainers (PG + Kafka + ES) 0건. 본 PR 의 핵심 가치(트랜잭션 정합성) 검증 수단 없음
